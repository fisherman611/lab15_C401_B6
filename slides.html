1. Project Overview

1.1 Problem Statement

Bối cảnh Tuyển sinh

  • Lượng lớn câu hỏi lặp lại (FAQ) gây quá tải.
  • Truy vấn phức tạp (điều kiện xét tuyển, tổ hợp ngành).
  • Hỗ trợ 24/7 cho sinh viên mới là điều bắt buộc.

Vấn đề hiện hữu

  • Scale: Con người không thể đáp ứng trong mùa cao điểm.
  • Phân mảnh: Thông tin rải rác ở PDF, Web, Forms.
  • Consistency: Khó đảm bảo tính nhất quán của câu trả lời.
1. Project Overview

1.2 Proposed Solution

01

Knowledge-aware QA

Xây dựng RAG pipeline để truy xuất tài liệu liên quan, sinh câu trả lời grounded (không hallucinate).

02

Reasoning Support

Xử lý câu hỏi multi-hop: "Em có đủ điều kiện ngành X nếu điểm em như thế này không?"

03

Workflow Assistant

Hướng dẫn step-by-step: Nộp hồ sơ, đăng ký môn học, xử lý giấy tờ hành chính.

2. Enterprise Context

2.3 Risk Analysis

❗ Hallucination

Sai lệch thông tin tuyển sinh gây hậu quả nghiêm trọng về pháp lý và uy tín.

❗ Outdated Knowledge

Quy định học vụ thay đổi hàng năm, AI có thể trả lời theo dữ liệu cũ.

❗ Data Leakage

Vi phạm quyền riêng tư của sinh viên (PII) khi gửi dữ liệu lên Cloud.

4. Cost Analysis

4.1 Formula & 4.2 Phase Breakdown

Cost = Nquery × (Cretrieval + CLLM)

* Trong đó CLLM chiếm phần lớn tổng chi phí.

MVP Phase (~$500)

Hệ thống đơn giản: 1 model, chưa routing, chưa cache.

Growth Phase (~$4k)

Traffic cao (Peak), dùng Caching, Routing, Autoscaling.

Cost Driver 1: Token usage
Cost Driver 2: Query volume
Cost Driver 3: Model Size
5. Optimization Plan

5.1 Multi-level Optimization (Part 1)

LVL 1

Caching

Cache các câu hỏi FAQ phổ biến giúp giảm 40–60% số lượng gọi LLM đắt tiền.

LVL 2

Model Routing

Dùng máy học phân loại: Câu đơn giản dùng model nhỏ, câu khó mới dùng model lớn.

LVL 3

Context Opt.

Chunking thông minh, Top-k linh hoạt (k=3-5) và Prompt compression.

5. Optimization Plan

5.1 Multi-level Optimization (Part 2)

LVL 4: Retrieval Optimization

Hybrid Search (kết hợp Keyword BM25 + Dense Embedding vector) và Cross-encoder Reranking để đạt độ chính xác tối đa.

LVL 5: Data Pipeline

Tự động cập nhật kiến thức (Knowledge Auto-update), quản lý phiên bản (Version control) để chống Outdated knowledge.

6. Reliability Plan

6.1 HA & 6.2 Queue System

High Availability

  • Auto-scaling trên Kubernetes clusters.
  • Load balancer phân phối tải đa điểm.
  • Multi-instance cho API layer.

Queue-based Processing

User → Queue → Worker → LLM

Ngăn chặn việc overload hệ thống khi traffic spike đột biến trong mùa tuyển sinh.
