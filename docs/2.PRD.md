# PRD — AI Alumni Networking Platform

## 1. Tổng quan sản phẩm

Nền tảng kết nối sinh viên và cựu sinh viên được tăng cường bởi AI. Chuyển đổi trải nghiệm networking từ "ngẫu nhiên" sang "có định hướng và phù hợp ngữ cảnh" tại các sự kiện alumni offline.

**Người dùng mục tiêu**: Sinh viên đang học, sinh viên mới tốt nghiệp, chuyên gia cấp trung tham gia demo sự kiện.

**Mục tiêu bản này**: Demo tập trung vào năng lực AI cốt lõi (Onboarding, Matching, Face Scan, Icebreaker), ưu tiên tốc độ xây dựng hơn độ đầy đủ vận hành production.

---

## 2. User Stories & Flows

---

### US-01 — Đăng ký & Đăng nhập

**Là** người dùng mới,
**Tôi muốn** đăng ký tài khoản bằng email/password hoặc Google/LinkedIn,
**Để** truy cập hệ thống với vai trò Alumni mặc định.

**Flow**:
```
User điền form đăng ký (email + password)
  → Hệ thống validate email chưa tồn tại
  → Hệ thống hash password (bcrypt cost 12)
  → Hệ thống tạo tài khoản với roles: ["alumni"]
  → Hệ thống tự động trigger AI Onboarding
```

**Acceptance Criteria**:
- Email trùng → lỗi rõ ràng, không tạo tài khoản
- Đăng nhập sai 5 lần / 15 phút → khóa 30 phút

---

### US-02 — AI Onboarding (Tạo hồ sơ qua trò chuyện)

**Là** người dùng mới vừa đăng ký,
**Tôi muốn** tạo hồ sơ qua cuộc trò chuyện ngắn với AI,
**Để** không phải điền form dài và vẫn có hồ sơ đầy đủ.

**Flow**:
```
Hệ thống tự động bắt đầu chat session (LangGraph)
  → AI sinh câu hỏi 1 (dựa trên missing fields)
  → User trả lời tự nhiên (tiếng Việt hoặc tiếng Anh)
  → AI extract structured fields từ câu trả lời
  → [Lặp lại tối đa 5 câu hỏi]
  → AI hiển thị profile preview để user review
  → User confirm (hoặc chỉnh sửa trực tiếp)
  → Hệ thống lưu profile vào DB
  → Hệ thống tạo vector embedding async (≤5 phút)
```

**Extracted fields**: `major`, `skills[]`, `experience`, `networking_goals`, `interests[]`

**Acceptance Criteria**:
- Tối đa 5 câu hỏi, AI tự skip câu hỏi nếu field đã được extract
- User có thể bỏ qua → tạo profile thủ công qua form
- Embedding sẵn sàng trong Matching Engine trong ≤5 phút

---

### US-03 — Smart Matching (Gợi ý kết nối tại sự kiện)

**Là** người tham dự sự kiện đã check-in,
**Tôi muốn** nhận gợi ý những người nên gặp,
**Để** mở rộng mạng lưới một cách có chất lượng.

**Flow**:
```
User truy cập trang gợi ý tại sự kiện
  → Hệ thống query Matching Engine
  → Matching Engine tính điểm hybrid:
      embedding_score × 0.6 + rule_score × 0.4
  → Hệ thống trả về top-10 gợi ý (≤3s)
  → User xem profile + điểm tương thích + lý do
  → User có thể: gửi kết nối / đánh dấu "không phù hợp"
```

**Scoring formula**:
- `embedding_score` = cosine_similarity(vector_A, vector_B) × 100
- `rule_score` = base(50) + skill_overlap(0–20) + intent_complementarity(0–20) + same_industry(0–10) − interaction_penalty

**Acceptance Criteria**:
- Chỉ gợi ý users đã check-in tại sự kiện đó
- Không gợi ý lại kết nối đã thiết lập
- Không gợi ý user đã bị đánh dấu "không phù hợp" tại event đó
- Kết quả cache Redis TTL 5 phút, refresh mỗi 5 phút

---

### US-04 — Scan to Connect (Nhận diện khuôn mặt)

**Là** người tham dự sự kiện,
**Tôi muốn** quét khuôn mặt người tôi muốn kết nối,
**Để** nhận diện và kết nối ngay lập tức mà không cần tìm kiếm thủ công.

**Flow**:
```
User mở camera "Scan to Connect" (mobile)
  → On-device: ML Kit detect face → TFLite MobileFaceNet → 512-dim vector
  → Gửi vector lên server (không gửi ảnh gốc)
  → Server query pgvector cosine similarity (≤1s)
  → Hiển thị profile tóm tắt ngay lập tức
  → [Song song] Matching Engine tính điểm (≤3s)
  → [Song song] Icebreaker Engine sinh 3 câu gợi ý (≤5s)
  → Progressive loading: cập nhật UI khi từng service hoàn thành
  → User gửi kết nối trực tiếp từ màn hình này
```

**Privacy rules**:
- Chỉ nhận diện nếu người được quét: `opt_in = true` AND `anonymous_mode = false`
- Không lưu ảnh gốc, chỉ lưu vector 512-dim (AES-256 encrypted)
- Rút consent → xóa face data trong ≤24h
- Thất bại 2 lần → gợi ý tìm theo tên/QR code

### US-05 — Realtime Suggestion (Demo giản lược)

**Là** người tham dự sự kiện đang diễn ra,
**Tôi muốn** nhận gợi ý kết nối được cập nhật liên tục,
**Để** không bỏ lỡ cơ hội gặp đúng người.

**Flow (Demo)**:
```
User chủ động bấm "Refresh suggestions"
  → Hệ thống tính lại top-10 suggestions theo dữ liệu mới nhất
  → Cập nhật UI ngay trên màn hình hiện tại
```

**Acceptance Criteria**:
- Không yêu cầu scheduler định kỳ và không bắt buộc WebSocket trong bản demo
- User có thể refresh thủ công và thấy kết quả mới trong cùng phiên

### US-06 — LLM Icebreaker (Câu mở đầu cuộc trò chuyện)

**Là** người tham dự sự kiện,
**Tôi muốn** nhận gợi ý câu mở đầu phù hợp khi gặp một người cụ thể,
**Để** tự tin hơn khi tiếp cận.

**Flow**:
```
User xem profile của người được gợi ý
  → Icebreaker Engine (LangGraph RAG pipeline):
      [retrieve_node] → query event knowledge base
      [grade_docs_node] → lọc chunks liên quan
      [generate_node] → GPT-4o sinh 3 câu icebreaker
      [validate_node] → kiểm tra đúng 3 câu, đúng ngôn ngữ
  → Trả về 3 câu gợi ý (≤5s)
  → User đánh giá "không phù hợp" → regenerate (tối đa 3 lần/cặp/event)
```

**Knowledge base sources**: Event agenda, speaker bios, common skills/interests từ Matching Engine

**Acceptance Criteria**:
- Luôn trả về đúng 3 câu
- Ngôn ngữ theo `language_preference` của user (vi/en)
- Không hiển thị câu gợi ý cho người được đề cập

---

### US-07 — Quản lý hồ sơ & Quyền riêng tư (Demo)

**Là** người dùng,
**Tôi muốn** kiểm soát thông tin hiển thị cho người khác,
**Để** bảo vệ quyền riêng tư.

**Privacy settings**:

| Setting | Hiển thị cho |
|---|---|
| `public` | Tất cả người dùng |
| `event_only` | Chỉ người cùng check-in sự kiện |
| `connections_only` | Chỉ kết nối đã accepted |

**Anonymous mode**:
```
User bật anonymous mode
  → Chỉ hiển thị initials + industry trong gợi ý
  → Tắt Face Recognition (không thể bị nhận diện)
  → Profile embedding vẫn dùng trong Matching (tính điểm ẩn danh)
```

**Demo privacy baseline**:
- Không lưu ảnh gốc
- Cho phép bật/tắt anonymous mode
- Cho phép bật/tắt consent nhận diện khuôn mặt
